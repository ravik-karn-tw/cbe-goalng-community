### Why go

- Easy to learn, Expressiveness, Readable => Simple & elegant
- Fast build and portable binary
- Orthogonal features set
- It doesn't compete with features
- Adding features will not make go better, It will just make it bigger
- Widely used in open source projects - k8s, docker, terraform, Cockroach DB ...

### Things that Go does not have

- implicit numeric conversions
- classes
- method overloading
- operator overloading
- default parameters
- <span style="color:orange">generics</span>
- exceptions
- macros
- function annotations
- thread-local storage

### Language construct - Data types
Primitive Types
- bool
- string
- int  int8  int16  int32  int64
- uint uint8 uint16 uint32 uint64 uintptr
- byte // alias for uint8
- rune // alias for int32, represents a Unicode code point
- float32 float64
- complex64 complex128

---

### Language construct - Data types
Composite Types
- Non-Reference Types
    - Arrays
    - Structs
- Reference Types
    - Slices
    - Maps
    - Channels
    - Pointers
    - Function/Methods
- Interface
